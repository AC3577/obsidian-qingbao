# Knowledge Base Schema

这是 LLM 维护知识库的规范文件。每次操作前必须读这里。

## 内容方向（四大主题）

所有采集和编译都围绕以下四个方向：
1. **自我提升文化** — 成长焦虑、习惯养成、自我认知、效率方法论
2. **赚钱** — 副业、职场晋升、财务自由、AI赚钱机会
3. **职场心理健康** — 职业倦怠、工作生活平衡、职场人际、情绪管理
4. **新时代灵性 + 实用主义** — 玄学/灵性的现代解读、冥想、能量、心理学与灵性的交汇

**排除**：地缘政治、娱乐八卦、技术教程（工具类）

## 目录结构

```
knowledge-base/
├── SCHEMA.md          ← 本文件，LLM 的操作规范
├── raw/               ← 原始采集文件（只读，不修改）
│   ├── reddit/        ← Reddit 研究报告
│   ├── xhs/           ← 小红书采集
│   ├── intel/         ← 晨报/晚报情报
│   ├── bilibili/      ← B站内容
│   └── 2026-*/        ← 按月归档的其他素材
├── wiki/              ← LLM 维护的知识库（LLM 写，人类读）
│   ├── index.md       ← 主题索引（每次 ingest 后更新）
│   ├── log.md         ← 操作日志（append-only）
│   └── _topics/       ← 每个主题一个目录
│       └── {主题名}/
│           ├── _L0.md      ← 一句话定义（不超过 30 字）
│           ├── _L1.md      ← 概览（300-500字，多次 ingest 追加）
│           ├── 核心数据.md  ← 数字/热度/互动量（追加）
│           ├── 痛点词表.md  ← 用户原声（追加）
│           ├── 选题建议.md  ← 可执行选题方向（追加）
│           ├── 金句摘录.md  ← 高传播力表达（追加）
│           └── 反向链接.md  ← 关联主题（[[主题名]]格式）
└── outputs/           ← query 产出归档
```

## Ingest 规范

新素材进入 raw/ 后，编译器自动执行：
1. 判断属于哪些主题（优先归入现有主题，确实新内容才建新主题）
2. 每个主题更新：_L1.md（概览追加）+ 相关子文件追加
3. 更新 index.md + 追加 log.md

**追加不覆盖**：同一主题的文件只追加新内容，不重写已有内容。

## Query 规范

- 读 index.md 确定相关主题 → 读完整主题内容 → 综合回答
- 有价值的回答用 `kb-query.py --save` 存入 outputs/
- 好的分析可以手动归档回对应主题文件

## Lint 规范（每周一次）

检查项：
- 有 _L0.md 但缺少其他文件的主题 → 数据空洞
- 反向链接指向不存在主题 → 引用断裂
- 超过 30 天未更新的主题 → 标记 [stale]
- 主题名与四大方向完全无关 → 标记待审核

## 主题命名规范

- 2-6 个中文字，名词短语
- 与四大方向挂钩（可以更具体，不要过于泛化）
- 例：✅ 副业焦虑、冥想实用化、职场倦怠信号
- 例：❌ 内容、生活、工作（太泛）
